### Model Description

A Random Forest classifier trained to predict cryptocurrency volatility spikes in real-time. The model identifies periods of abnormal price volatility (exceeding the 95th percentile threshold) using engineered features from streaming market data. This model is the final output of the `models/train.py` script.

### Intended Use

**Primary Use Case:**  
Real-time alerting system for cryptocurrency trading platforms to detect abnormal market volatility.

**Intended Users:**
- Cryptocurrency traders
- Risk management teams
- Algorithmic trading systems
- Market surveillance systems

**Out-of-Scope Uses:**
- Direct trading decisions without human oversight
- Use with cryptocurrencies other than Bitcoin (model trained on BTC/USD)
- Regulatory compliance (model not validated for regulatory purposes)

## Training Data

**Source:** Kraken Exchange WebSocket API (simulated)
**Symbol:** BTC/USD (Bitcoin/US Dollar)  
**Period:** November 2025  
**Total Records:** 32,233 observations  
**Train/Test Split:** 80/20 (stratified by the notebook, loaded by `train.py`)

**Training Set:**
- Size: 25,789 samples
- Positive class (spikes): 1,290 samples (5.0%)

**Test Set:**
- Size: 6,448 samples  
- Positive class (spikes): 322 samples (4.99%)

**Feature Count:** 13 engineered features

### Label Definition

**Volatility Spike:**  
Binary classification where `1` indicates a volatility spike (abnormal volatility) and `0` indicates normal market conditions.

**Threshold (τ):** 0.041799 (95th percentile of `volatility_60s`)

**Train/Test Split Quality:**
- An 80/20 stratified split was created by the `03_model_training.ipynb` notebook and saved to disk.
- The final `train.py` script uses these pre-split and validated datasets to ensure consistency.
- Data drift analysis was performed, and the report is available at `reports/evidently_data_drift_report.html`.

## Features

### Engineered Features (13 total)

**Price Features (3):**
- `price`: Current mid-market price (BTC/USD)
- `bid`: Best bid price
- `ask`: Best ask price

**Volatility Features (3):** ⭐ Most Important
- `volatility_30s`: 30-second rolling standard deviation
- `volatility_60s`: 60-second rolling standard deviation  
- `volatility_120s`: 120-second rolling standard deviation

**Return Features (3):**
- `return_10s`: 10-second price return
- `return_30s`: 30-second price return
- `return_60s`: 60-second price return

**Market Microstructure Features (4):**
- `spread`: Bid-ask spread
- `intensity_30s`: 30-second trade intensity
- `intensity_60s`: 60-second trade intensity
- `intensity_120s`: 120-second trade intensity

## Ethical Considerations

### Potential Biases
- **Temporal Bias:** Model trained on a specific, short time window in November 2025. Market dynamics will change.
- **Symbol Bias:** Only trained on BTC/USD; will not generalize to other assets without retraining.
- **Market Condition Bias:** The training data may not include "black swan" or extreme market events.

### Limitations
1. **Recency Bias:** Assumes recent market patterns will persist.
2. **Single Asset:** Not tested on altcoins or other trading pairs.
3. **No Fundamental Data:** Ignores news, sentiment, and other external factors.

### Risks
- **Overconfidence:** High precision may lead to over-reliance. The model is not infallible.
- **Adversarial Conditions:** The model has not been tested against intentionally manipulative market behavior.

## Recommendations

### Deployment Guidelines
1. **Review Configuration:** Check `config.yaml` for default parameters and `docs/feature_spec.md` for feature definitions.
2. **Set Environment Variables:** Use `.env` file for deployment-specific settings (Kafka servers, MLflow URI, API keys).
3. **Use as an Alerting System:** Predictions should trigger alerts for human review, not automated trades.
4. **Continuous Monitoring:** Implement drift detection on model predictions and input features (see `config.yaml` monitoring section).
5. **Regular Retraining:** Retrain the model weekly, or more frequently if market volatility increases.

### Retraining Schedule
- **Frequency:** Weekly (configurable in `config.yaml` - `monitoring.model_monitoring.retrain_schedule`)
- **Triggers:** 
  - Production F1 score drops below 0.95 (threshold in `config.yaml`)
  - Significant data drift is detected in key volatility features (Evidently threshold: 0.05)
